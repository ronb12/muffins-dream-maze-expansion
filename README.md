# Muffin's Maze Adventure (Web)

First-person dream maze ‚Äî 100 worlds, story chapters, dream store, touch controls, PWA install.

**Live (Vercel):** https://muffins-dream-maze-expansion.vercel.app  
**Repo:** https://github.com/ronb12/muffins-dream-maze-expansion  
**GitHub `main`:** multi-page site (`index`, `play`, `console`, `worlds`, `story`, `store`, `settings`)

|------|-------|------------|

Shared top nav: `js/site-nav.js` on every page. Game logic runs on all pages via `app.js` + hidden stubs (`js/app-stubs.js`); only **Play** runs the 3D loop.

## Run locally

```bash
python3 -m http.server 9876
```

Open [http://127.0.0.1:9876/](http://127.0.0.1:9876/) (use your LAN IP on a phone).

| **Neon Postgres** | **Not linked** ‚Äî no Neon project named for this game; no `DATABASE_URL` on Vercel. Progress saves in browser `localStorage` only. |

Redeploy from this folder:

```bash
npx vercel --prod
```

`vercel.json` sets `cleanUrls: true` so `/play` serves `play.html`.

## Music (GarageBand ‚Üí original soundtrack)

1. **Play now:** placeholder MP3s are already in `audio/` (generated by `scripts/build-dream-audio.py`).
2. **Your GarageBand tracks:** compose in GarageBand, export MP3s with the filenames in `audio/GARAGEBAND.md`, drop into `audio/`, refresh the game.

```bash
./scripts/open-garageband-audio.sh   # opens GarageBand + audio folder
python3 scripts/build-dream-audio.py # regenerate dev placeholders
```

## Features

- Tutorial, play-focus layout, world map, daily dream
- Original music + SFX (`js/audio-manager.js`)
- Mobile touch overlay, haptics, wake lock, settings (sensitivity, save export/import)
- PWA (`manifest.json`, `sw.js`)
